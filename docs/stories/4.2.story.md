# Story 4.2: Integrate Core Services with Task Manager

## Status: Draft

## Story

**As a developer,** I need the import and download services to report their status to the new Task Management service to handle partial failures gracefully.

## Acceptance Criteria (ACs)

* AC1: Playlist imports and downloads must create a "parent" task. The parent task's progress should reflect the completion of its "child" video tasks.
* AC2: If some videos in a playlist download fail, the parent task should be marked as "Completed with errors."
* AC3: The download service must update the task's progress and final status in the database.

## Tasks / Subtasks

**Backend (`Main Process`)**

* [ ] **Task 1 (AC: 1):** Refactor the playlist import and download services (`playlist:import` and `download:start` handlers).
    * [ ] Subtask 1.1: Before adding a job to the `p-queue`, the services must first call the `TaskManagementService` to create the appropriate parent and/or child task entries in the database.
* [ ] **Task 2 (AC: 3):** Modify the core download/import logic (the functions that are executed by `p-queue`).
    * [ ] Subtask 2.1: This logic must now accept a `taskId` as a parameter.
    * [ ] Subtask 2.2: Throughout its execution, it must call the `TaskManagementService` to report progress updates (e.g., percentage complete) and status changes (e.g., `IN_PROGRESS`, `COMPLETED`, `FAILED`).
* [ ] **Task 3 (AC: 2):** Implement logic within the `TaskManagementService` to handle the completion of a child task. When a child task finishes, the service should check the status of all sibling tasks to determine if the parent task's status needs to be updated (e.g., to `Completed with errors` or `Completed`).

## Dev Technical Guidance

* **Refactoring Focus:** This story does not add new user-facing features but is a critical internal refactor. The goal is to make all background jobs stateful and persistent by using the `TaskManagementService` created in Story 4.1.
* **Data Flow:** The new flow for a background job is:
    1.  IPC handler is called.
    2.  Handler calls `TaskManagementService` to create a persisted task record.
    3.  Handler adds the job to the `p-queue`, passing the new `taskId` to it.
    4.  The job function executes, and as it does, it calls `TaskManagementService` to update the status and progress of its `taskId`.
* **Parent Task Aggregation:** For playlist downloads (parent tasks), the `TaskManagementService` needs to aggregate the status of its children. For example, the parent's progress can be calculated as `(number of completed child tasks / total number of child tasks) * 100`. When the final child task completes, the service should determine the parent's final status (`Completed` vs. `Completed with errors`).
